What is Kryon?

Kryon is a high-level transpiler that brings 90s simplicity to the modern era. By combining the logic of Limbo with a declarative, Tk-compatible syntax, Kryon allows you to build sophisticated interfaces that talk directly to distributed hardware via the 9P protocol.

Stop wrestling with bloated frameworks. Kryon uses a clean, block-based architecture for UI and system modules.


Features

  • Pure Efficiency: Transpiles directly to optimized system code.
  • Native 9P: File-sharing and network resources are handled as local objects.
  • Tk Styling: Classic, predictable, and incredibly fast UI rendering.
  • Unified Modules: Access every system resource through a single, simple interface.
  • Declarative Syntax: Write less, do more with block-based UI definitions.

Code Example

Simple button with callbacks and Tk-compatible styling:

// Kryon Button Example
// Demonstrates button with callbacks and Tk-compatible styling

fn handleClick() {
    sys->print("Button clicked!\n");
}

Window {
    title = "Button Demo"
    width = 600; height = 400
    bg = "#191919"

    Center {
        Button {
            text = "Click Me!"
            fg = "pink"; bg = "#404080"
            font = "Helvetica 14 bold"
            padx = 20; pady = 10
            relief = "raised"; borderwidth = 2
            onClick = handleClick
        }
    }
}
